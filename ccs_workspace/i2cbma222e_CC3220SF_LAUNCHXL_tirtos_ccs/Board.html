CC3220SF LAUNCHXL Settings and Resources

The CC3220SF LAUNCHXL board contains a CC3220SF device.

Jumper Settings

  • Set RXD << and TXD >> to provide UART communication via the onboard USB debugger.
  • If you use a BoosterPack with this board, the BoosterPack fits over J1 and J2.

Board resources used in driver examples

The following table lists the Board names of the resources used by the Sensor and Actuator Plugin examples, and their corresponding pins. Note that there may be other resources configured in the board files that are not used by any example. Only the example resources are listed in the table.

Board Resource Pin Remarks
Board_LED0 P64 (LED EN must be jumpered)
Board_LED1 P01 (LED EN must be jumpered)
Board_LED2 P02 (LED EN must be jumpered)
Board_PWM0 P01 Board_LED1,Board_LED2 is PWM LED we need PWM0 to enable PWM LED
Board_PWM1 P02 Board_LED2 is PWM LED we need PWM1 to enable PWM LED
Board_BUTTON0 P15 (SW2)
Board_BUTTON1 P04 (SW3)
Board_I2C0 P01, P02 Also stated as Board_I2C_TMP, Board_I2C_OPT in Board.h
Board_UART0 P57, P55 UART provided through emulation (RX and TX must be jumpered) Also stated as Board_UART_PGA460 in Board.h

Peripherals Used

The following list shows which CC3220SF peripherals are used by Sensor and Actuator Plugin applications.

  • TI-RTOS Kernel (SYS/BIOS). Uses POSIX API for timers,semaphores,Tasks.The TI-RTOS Kernel manages the interrupt controller statically without an interrupt dispatcher.

  • FreeRTOS Kernel). Uses POSIX API for timers,semaphores,Tasks.

  • Drivers.
    • I2C: The I2C driver is configured on EUSCI_B0 to support various BoosterPacks.
    • PWM: The PWM driver used within the LED module, LED,Button Example application, the onboard RGB LED (G:P2.1 and B:P2.2). These pins are configured for the PWM driver. While these pins can also be used by the GPIO driver, your application’s board file must ensure that the pins are not simultaneously used by the GPIO and PWM drivers.
    • GPIO: The GPIO drivers is used for handling Push Button Interrupts, Interrupts from Sensors, GPIO outputs for LED.
    • UART: The UART driver uses UART0, which is attached to the onboard emulator to facilitate serial communications.The UART driver(UART1) is also used for interfacing BoosterPacks like BOOSTXL-PGA460
